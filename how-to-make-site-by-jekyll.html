제킬(Jekyll)을 사용하여 페이지 유지하기

1. Jekyll

이 정적 사이트(Static Stie)

1.1. Jekyll 설치하기

Jekyll은 Ruby를 사용한다. 설치 또한 Ruby의 패키지 설치 관리자인 gem을 사용하기 때문에 먼저 Ruby의 설치가 필수적이다.

  • 루비 설치하기 : 설치프로그램 다운로드 하기

1.1.1. Ruby 설치하기

설치에 gem을 사용해야 하기 때문에 Ruby를 설치해야 한다.

$ gem install jekyll bundler
$ jekyll new my-awesome-site
$ cd my-awesome-site
$ bundle exec jekyll serve
# => Now browse to http://localhost:4000

위에 보면 $ gem install jekyll bundler을 적어서 install 하는데 이는 jekyll과 bundler 둘 다 install 된다는 것이다.

위를 통해서 사이트의 구성 file들을 생성하면 여러가지 알수 없는 파일들이 생성된다. 하나씩 살펴보도록 하자.

■ Gemfile, Gemfile.lock

두 파일 모두다 Static Site Generator를 위한 파일이다. Site의 구성 파일은 아니고 생성하기 위한 옵션이나 Dependencies를 설정하는 것이라고 보는게 올바르다.

2. 사이트 구성하기

기본적으로 글들은 post폴더에 들어간다. 따라서 별다른 새로운 페이지를 구성하는 것이 아니라면 post폴더에 글들을 넣으면 된다.

아 그리고 기본설정이 file의 YYYY-MM-dd 기준으로 Perma-Link가 생성되는 듯 하다. 따라서 글을 적을 때 파일 명에 유의해서 만들어야 할 것 같다.

주의사항

  1. File명칭을 적을 때 작성일이나 수정일을 신경써서 적어야 한다.
  2. permalink를 만들어줄 때는 당연히 신경써서 만들어 줘야 하구….
